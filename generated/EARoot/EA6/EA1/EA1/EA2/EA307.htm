EAO provides thread-safe database transactions for JPA persistent objects. The objects to be persisted must implement the {@link IEntity} interface. All method calls will open a database transaction if needed. However, you can control when the transaction is closed. Each method has 2 forms: <br /></p><p>#lt;h3#gt;Method Scope#lt;/h3#gt;  Method scope transaction close the transaction when the method exits. It's the easiest scope to use as you do not need to explicitly close a transaction. <br /></p><p>#lt;pre#gt; EAO.insert(entity) //transaction starts and ends inside the method EAO.insert(entity, true) // same as above<br /></p><p>#lt;/pre#gt; <br /></p><p>#lt;h3#gt;Extended Scope#lt;/h3#gt;  Make several methods calls within a single transaction <br /></p><p>#lt;pre#gt; EAO.insert(entity, false); EAO.insert(entity2, false);  You will need to close the transaction either EAO.endTransaction(); EAO.insert(entity2, true) would've also close the transaction <br /></p><p>#lt;/pre#gt; <br /></p>
